# 08 — References

## Role

Citations must be complete, correctly ordered, and in Nature house style — in both the main
text and the Supplementary Information.

## In-text citation

- **Numerical superscripts in order of first appearance.** The first source cited is ¹, the
  next new source ², and so on — no gaps, no out-of-order numbers.
- Renumber after any reordering of the text (a frequent revision error).

## Reference format (Nature style)

Each entry needs: **authors (initials)**, **title**, **journal/publisher** (*italic*),
**volume** (**bold**), **page range**, **year**.

- Journal article (shape):
  > Author, A. B. *et al.* Title of the article. *Journal Abbrev.* **Vol**, start–end (Year).
- Dataset / report (shape):
  > Author, A. *et al.* Dataset or report title. Repository/Publisher https://doi.org/… (Year).

Datasets and reports **must** include the repository name, DOI, and year. Use *et al.* per
the journal's author-count convention. (Journal-name abbreviation style can be left as-is —
the editor normalises it.)

## Supplementary references (baked in)

- The SI has its **own reference list at the end of the Supplementary Information**,
  **numbered from 1**, and **self-contained** — it must not cross-reference the main list.
- A source cited in **both** the main text and the SI appears in **both** lists (duplicated),
  never as "see main reference [n]".

## Common revision errors

- Out-of-order or gapped numbering after edits → re-verify the sequence.
- Missing DOI/year on dataset and software references.
- SI references pointing back to the main list instead of standing alone.

## Self-check

Citations numbered in first-appearance order with no gaps? Every entry complete (authors,
title, italic journal, bold volume, pages, year)? Datasets/code with repository + DOI + year?
SI reference list self-contained, numbered from 1, shared sources duplicated?
